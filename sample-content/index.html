💰 x402 Payment Gateway

Sample Origin

This page is served through an Amazon CloudFront distribution protected by the x402 payment protocol. AI agents and bots must pay with USDC stablecoins to access gated content.

How It Works

  1. AWS WAF classifies incoming traffic (human, verified bot, unverified bot).
  2. Requests matching a priced route receive an HTTP 402 response with payment requirements.
  3. The client signs a payment and resends the request with an X-PAYMENT header.
  4. Lambda@Edge verifies the payment via a facilitator, then forwards the request to the origin.
  5. On a successful origin response, the payment is settled on-chain.

Testing the Payment Flow

Use curl to test against your CloudFront distribution:

# 1. Request gated API content (expect 402 Payment Required)
curl -i https://<your-distribution>.cloudfront.net/api/sports.json

# 2. The 402 response body contains payment requirements:
#    { "x402Version": 1, "accepts": [{ "scheme": "exact", ... }] }

# 3. After signing a payment, include the header:
curl -i -H "X-PAYMENT: <signed-payment>" \
  https://<your-distribution>.cloudfront.net/api/sports.json

# Try other endpoints too:
curl -i https://<your-distribution>.cloudfront.net/api/fashion.json
curl -i https://<your-distribution>.cloudfront.net/articles/politics.html
